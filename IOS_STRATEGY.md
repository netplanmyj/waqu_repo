# iOS版リリース戦略まとめ

**最終更新**: 2025年10月30日

---

## 📊 戦略の結論

### ✅ 推奨戦略: **無料・広告なし**

Android版と同様に、iOS版も**無料・広告なし**で公開します。

---

## 🎯 この戦略を選ぶ理由

### 1. App Store審査の観点

| 項目 | 無料・広告なし | 課金・広告あり |
|-----|--------------|--------------|
| **審査難易度** | ⭐⭐（低い） | ⭐⭐⭐⭐（高い） |
| **審査期間** | 1-2週間 | 2-4週間 |
| **追加審査項目** | なし | 多数あり |
| **リジェクトリスク** | 低い | 高い |

**無料・広告なしの利点**:
- ✅ シンプルな構成で審査が通りやすい
- ✅ 広告SDKの審査が不要
- ✅ 課金システムの実装・テスト不要
- ✅ プライバシー要件が最小限

**課金・広告ありの課題**:
- ❌ App Store In-App Purchase の実装必須（複雑）
- ❌ 広告SDKのプライバシー審査が厳格
- ❌ ATT（App Tracking Transparency）対応必須
- ❌ 収益化の実装とテストに時間がかかる

---

### 2. 開発速度の観点

**無料・広告なし**:
```
開発期間: 2-4週間
└─ Week 1: 環境セットアップ
└─ Week 2: ビルド・テスト
└─ Week 3-4: App Store審査待ち
```

**課金・広告あり**:
```
開発期間: 6-8週間
└─ Week 1-2: 環境セットアップ + マネタイズ実装
└─ Week 3: 課金・広告のテスト
└─ Week 4: プライバシー・ATT対応
└─ Week 5-6: ビルド・テスト
└─ Week 7-8: App Store審査待ち（厳格化）
```

**時間短縮**: 4週間の差

---

### 3. Android版との一貫性

| 項目 | Android版 | iOS版（推奨） |
|-----|-----------|-------------|
| **価格** | 無料 | 無料 |
| **広告** | なし | なし |
| **課金** | なし | なし |
| **価値提供** | シンプルなツール | シンプルなツール |

**一貫性の利点**:
- ✅ ユーザーに同じ体験を提供
- ✅ ブランドイメージの統一
- ✅ プロモーション文言が同じ
- ✅ サポート対応がシンプル

---

### 4. 水道業界への適合性

**公共性の高い業務ツール**:
- 水道は公共インフラ
- 自治体職員が主なユーザー
- 無料・広告なしが信頼感につながる
- 業務ツールに広告は不適切

**ユーザーの期待**:
```
❌ 広告が表示される → 業務中の妨げになる
❌ 課金が必要 → 予算申請が必要で導入ハードルが高い
✅ 無料・広告なし → すぐに導入できる、信頼性が高い
```

## 🚀 iOS版リリースへのロードマップ

### Phase 1: 環境準備（1週間）

```
□ Apple Developer Program登録（$99/年）
□ Mac環境確保（実機 or クラウド）
□ Xcode最新版インストール
□ CocoaPods セットアップ
```

**コスト**: $99 + Mac環境費用（$0-1,200）

---

### Phase 2: アプリ設定（1週間）

```
□ pubspec.yaml で ios: true に変更
□ flutter pub run flutter_launcher_icons 実行
□ Firebase iOS設定（GoogleService-Info.plist）
□ Info.plist に Google Sign-In URL Scheme追加
□ Bundle Identifier 設定（jp.netplan.ios.waquRepo）
```

**詳細**: `IOS_SETUP.md` 参照

---

### Phase 3: ビルドとテスト（1週間）

```
□ iOS Simulator でテスト
□ iPhone実機でテスト（推奨）
□ Google Sign-In 動作確認
□ メール送信機能確認
□ 設定・履歴機能確認
□ クラッシュがないか確認
```

---

### Phase 4: App Store準備（1週間）

```
□ スクリーンショット撮影（6.7インチ、3-5枚）
□ App Store掲載情報作成
  - アプリ名（30文字以内）
  - サブタイトル（30文字以内）
  - 説明文（4000文字以内）
  - キーワード（100文字以内）
□ プライバシーポリシーURL確定
□ デモアカウント準備（waqu.reviewer@gmail.com）
```

**詳細**: `APP_STORE_CHECKLIST.md` 参照

---

### Phase 5: 審査申請（1-2週間）

```
□ App Store Connect で新規App作成
□ ビルドをアップロード
□ TestFlight でテスト（任意）
□ 審査ノートにデモアカウント情報記載
□ 審査申請
□ 審査状況を定期的に確認
```

**審査期間**: 通常1-2週間

---

### Phase 6: 公開（即日）

```
□ 承認通知受領
□ 「バージョンをリリース」をクリック
□ App Store に公開（数時間以内）
□ 動作確認
□ 🎉 iOS版リリース完了！
```
