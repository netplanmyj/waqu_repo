# デモ動画でのプライバシー保護ガイド

OAuth審査用デモ動画で個人情報を保護するための方法です。

---

## 🔒 基本方針

### Google OAuth審査で必要なこと

```
✅ 必要: 機能の実証
  - OAuth同意画面が表示される
  - Gmail APIでメール送信できる
  - 送信されたメールを確認できる

❌ 不要: 個人情報の開示
  - 実際のメールアドレス
  - 実際の氏名
  - その他の個人識別情報
```

### 重要な原則

> **「機能が動作していることを示せば十分」**  
> 実際のメールアドレスや個人情報を見せる必要はありません。

## 📧 メールアドレスの扱い方

### 方法1: 架空のメールアドレスを使用（推奨）

#### Googleアカウント
```
撮影用にデモ専用アカウントを作成:
メールアドレス: waqu.demo@gmail.com
または: waterquality.test@gmail.com

メリット:
✅ 編集不要
✅ そのまま公開できる
✅ セキュリティリスクなし
```

#### 送信先メールアドレス
```
架空のアドレスでOK:
- demo@example.com
- recipient@example.com
- test@example.com

理由:
- アプリの動作を示すのが目的
- 実際に受信する必要はない
- 審査担当者は機能を見るだけ
```

---

### 方法2: 既存アカウント＋編集時にモザイク処理

#### 撮影時
```
実際のアカウントでそのまま撮影
- your.account@gmail.com
- recipient@company.com
```

#### 編集時にモザイク処理
```
対象箇所:
1. Googleアカウント選択画面
   - メールアドレス部分をモザイク
   - アカウント名をモザイク

2. 設定画面
   - 送信先メールアドレスをモザイク
   - またはテキストオーバーレイで隠す

3. Gmail確認画面
   - 送信者アドレスをモザイク
   - 受信者アドレスをモザイク
```

#### モザイク処理の方法

**DaVinci Resolve**:
```
1. エフェクト → Blur → Gaussian Blur
2. マスクで該当箇所を選択
3. ぼかし強度: 50-100
4. トラッキング（画面が動く場合）
```

**iMovie**:
```
1. テキストオーバーレイで隠す
   例: "demo@example.com"
2. または黒い四角形で覆う
```

**Clipchamp**:
```
1. 図形 → 長方形を追加
2. メールアドレス部分に配置
3. 色: 黒または白
```

---

### 方法3: テキストオーバーレイで置き換え

#### 撮影後の編集で追加

```
実際のメールアドレスの上にテキストを重ねる:

before: your.real.email@gmail.com
after:  demo@example.com (テキストで上書き)

メリット:
✅ 自然な見た目
✅ 編集が簡単
✅ プライバシー完全保護
```

## 🎨 編集ツール別モザイク処理

### DaVinci Resolve（無料、推奨）

```
1. タイムラインで該当クリップを選択
2. エフェクト → Blur → Gaussian Blur をドラッグ
3. インスペクター → Blur → Power Window
4. マスク（Circle, Rectangle等）でメールアドレス部分を選択
5. Blur Amount: 50-100 に設定
6. トラッキング:
   - Tracker Palette → Track Forward
   - 画面が動く場合は自動追尾
```

### iMovie（Mac）

```
1. クリップを選択
2. テキストオーバーレイを追加:
   - テキスト → タイトル → Lower Third
   - テキスト: "demo@example.com"
   - 位置: メールアドレス部分
   - 背景: 黒または白（不透明度100%）

または

1. 図形（黒い四角）をメールアドレス部分に配置
2. 不透明度: 100%
```

### Clipchamp（オンライン）

```
1. タイムラインで該当部分を選択
2. テキスト → 追加
   - 内容: "demo@example.com"
   - フォント: Arial Bold
   - サイズ: メールアドレスと同じ
   - 位置: メールアドレスの上

または

1. 図形 → 長方形
2. メールアドレス部分に配置
3. 色: 黒または白
```

## 🆘 よくある質問

### Q1: 架空のメールアドレスで送信できますか？

**A**: はい、できます。

```
理由:
- アプリは入力されたアドレスに送信する
- 受信確認は審査に不要
- Gmail APIが動作することを示せば十分

ただし:
- Gmail確認シーンで送信済みメールを見せる必要がある
- 送信先が架空でも、送信済みフォルダには残る
```

---

### Q2: Googleアカウントの写真や名前は見えてもいいですか？

**A**: デモ用アカウントなら問題ありません。

```
推奨:
- アカウント名: Demo User, Test User
- プロフィール写真: デフォルトアイコン

実名・実写真の場合:
- モザイク処理を推奨
- またはテキストオーバーレイで隠す
```

---

### Q3: 送信先メールアドレスは実在する必要がありますか？

**A**: いいえ、架空でOKです。

```
審査で確認されること:
✅ アプリで送信先を設定できる
✅ Gmail APIで送信処理が実行される
✅ 送信済みフォルダにメールが残る

審査で確認されないこと:
❌ 送信先が実在するか
❌ メールが実際に受信されたか
```

---

### Q4: モザイク処理が面倒です。他の方法は？

**A**: デモ用Googleアカウントを作成するのが最も簡単です。

```
10分で完了:
1. Googleアカウント新規作成
2. メールアドレス: waqu.demo@gmail.com
3. アカウント名: Demo User
4. このアカウントでアプリにサインイン
5. 撮影

メリット:
✅ 編集作業ゼロ
✅ モザイク不要
✅ そのまま公開
```

---

### Q5: Gmail確認シーンで受信メールを見せる必要は？

**A**: いいえ、送信済みメールを見せれば十分です。

```
審査で重要なのは:
✅ ユーザー自身のGmailアカウントから送信できる
✅ Gmail APIのgmail.sendスコープが動作する

受信確認は不要:
❌ 送信先で受信する必要なし
❌ 受信トレイを見せる必要なし
```
