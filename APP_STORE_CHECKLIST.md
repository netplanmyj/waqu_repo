## 📋 4. よくあるリジェクト理由への対策

### Guideline 2.1 - Performance: App Completeness

❌ **リジェクト例**: アプリの機能が不完全、バグがある

✅ **対策**:
- [ ] すべての機能が実装済み
- [ ] デモアカウントで全機能をテスト済み
- [ ] クラッシュやフリーズがない
- [ ] エラーメッセージが適切に表示される

---

### Guideline 4.0 - Design: Minimum Functionality

❌ **リジェクト例**: アプリの機能が最小限すぎる、Webサイトをラップしただけ

✅ **対策**:
- [ ] 説明文でアプリの価値を明確に記載
- [ ] 水道業界向けの専門ツールであることを強調
- [ ] ネイティブUIを使用（Material Design）
- [ ] スクリーンショットで具体的な使用例を示す

**説明文に含める文言例**:
```
本アプリは水道事業体の業務効率化を目的とした専門ツールです。
日々の水質検査報告をモバイル端末から即座に送信できます。
従来の紙ベースやPC操作と比較して、作業時間を大幅に短縮できます。
```

---

### Guideline 4.3 - Design: Spam

❌ **リジェクト例**: 類似アプリが既に存在する、差別化されていない

✅ **対策**:
- [ ] アプリの独自性を説明文に記載
- [ ] 水道業界特化であることを明示
- [ ] ターゲットユーザーを明確にする

**独自性の説明例**:
```
【本アプリの特徴】
- 水道業界の業務フローに最適化
- 日本の水質基準項目に対応
- オフライン環境での入力に対応
- 自治体の報告フォーマットに準拠
```

---

### Guideline 5.1.1 - Legal: Privacy - Data Collection and Storage

❌ **リジェクト例**: プライバシーポリシーが不明確、データ収集の説明不足

✅ **対策**:
- [ ] プライバシーポリシーURLが有効
- [ ] Google Sign-In の使用を明記
- [ ] データの保存場所を説明（端末ローカル）
- [ ] App Store Connect の「プライバシー」セクション記入

**プライバシーポリシーに含める項目**:
```
1. 収集するデータ
   - Googleアカウント情報（メールアドレス、名前）
   - 入力された水質データ
   - 送信履歴

2. データの使用目的
   - メール送信機能の提供
   - 履歴表示機能の提供

3. データの保存場所
   - 端末のローカルストレージ（SharedPreferences）
   - Firebase Authentication（認証情報のみ）

4. データの共有
   - 第三者への共有なし
   - ユーザーが指定したメールアドレスにのみ送信

5. データの削除
   - アプリアンインストール時に自動削除
   - 設定画面から履歴を手動削除可能
```

---

### Guideline 2.3.1 - Performance: Accurate Metadata

❌ **リジェクト例**: スクリーンショットとアプリの実際の動作が異なる

✅ **対策**:
- [ ] スクリーンショットは実際のアプリから撮影
- [ ] 説明文とアプリの機能が一致
- [ ] 誤解を招く表現がない

## 📋 7. 審査状況の確認

### 審査の各段階

```
1. 「審査待ち」(Waiting for Review)
   - 審査キューに入った状態
   - 期間: 数時間〜数日

2. 「審査中」(In Review)
   - レビュアーが実際にアプリをテスト中
   - 期間: 1-3日

3. 「審査完了」
   - 承認 (Ready for Sale)
   - リジェクト (Rejected)
```

### リジェクト時の対応手順

1. **Resolution Center を確認**
   - App Store Connect → 該当App → Resolution Center
   - リジェクト理由を確認

2. **必要な修正を実施**
   - コード修正が必要な場合: 修正 → 再ビルド → 再アップロード
   - 説明文の問題: App Store Connect で編集

3. **再審査申請**
   - 修正内容を「審査ノート」に記載
   - 「審査に再提出」ボタンをクリック
